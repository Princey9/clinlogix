Problem

Healthcare IT systems generate large volumes of logs and structured data. Manually inspecting these files is time-consuming and error-prone, often delaying troubleshooting and data quality checks.

Solution

ClinLogix provides a lightweight CLI that:

  • Summarizes errors and warnings in log files
  • Validates FHIR JSON resources and reports conformance issues
  • Groups validation findings by category, theme, and severity with JSON path expressions
  • Adds best-effort line numbers to help you locate issues in large bundles
  • Supports JSON output and containerized execution via Docker

Demo Commands

Scan logs:

clinlogix scan demo-healthit.log
clinlogix scan demo-healthit.log --errors-only
clinlogix scan demo-healthit.log --json

Validate FHIR resources:

clinlogix validate examples/patient.json
clinlogix validate examples/patient-bad.json

(Screenshots of these commands are included in the project repository.)

Validating Synthea Bundles

Synthea bundles often depend on US Core and other implementation guide profiles. If the validation server does not have those packages installed, you will see errors like ‚ÄúUnable to resolve reference to profile ...‚Äù. ClinLogix now groups those under a profile resolution theme while preserving category grouping by severity/code/message, JSON path expressions, and best-effort line numbers to help you triage results.

If you encounter profile resolution errors, you can try a different validator:

  • Firely server (default): https://server.fire.ly
  • HAPI public R4 server: https://hapi.fhir.org/baseR4
  • A local validator instance with the required packages installed

Run with Docker

ClinLogix can be run without installing Rust by using Docker.

docker build -t clinlogix:latest .

docker run --rm -v "$PWD:/data" clinlogix:latest validate /data/examples/patient.json
docker run --rm -v "$PWD:/data" clinlogix:latest validate /data/examples/patient-bad.json
docker run --rm -v "$PWD:/data" clinlogix:latest scan /data/demo-healthit.log --json

Download

Option A: Build from source 

cargo build --release
./target/release/clinlogix --help

Option B: Download the pre-built binary from GitHub Releases.

macOS note:
This is an unsigned command-line tool. macOS may block it on first run. Remove quarantine and run from Terminal: 

chmod +x clinlogix-macos
xattr -d com.apple.quarantine clinlogix-macos
